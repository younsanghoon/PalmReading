# 웹사이트 표시 문제 해결 분석

## 문제 요약
GitHub Pages에 배포된 PalmReading 앱이 "성격 분석 테스트"로 표시되는 문제가 있었습니다. 이미지에서 볼 수 있듯이 웹사이트가 "성격 분석 테스트"라는 제목과 "애플리케이션이 로드되었습니다"라는 메시지를 표시하고 있었습니다.

## 원인 분석
1. **앱 메타데이터 불일치**: 
   - 앱의 제목, 설명 등의 메타데이터가 이전 프로젝트인 "성격 분석 테스트"로 설정되어 있었음
   - `client/src/App.tsx`, `client/index.html`, `client/public/manifest.json` 파일에 모두 이전 프로젝트 정보가 포함됨

2. **배포 설정 문제**:
   - GitHub Pages 배포 시 이전 프로젝트의 정보가 그대로 유지됨
   - gh-pages 브랜치에 이전 프로젝트 파일들이 남아있었을 가능성이 높음

## 해결 과정

### 1. 메타데이터 업데이트
다음 파일들의 메타데이터를 "손금 테스트"로 일관되게 변경했습니다:

1. **client/src/App.tsx**:
   - 제목을 "성격 분석 테스트"에서 "손금 테스트"로 변경
   - 설명을 "애플리케이션이 로드되었습니다"에서 "손금으로 보는 나의 운명"으로 변경

2. **client/index.html**:
   - `<title>` 태그를 "성격 테스트 센터 - AI 성격 분석"에서 "손금 테스트 - AI 손금 분석"으로 변경
   - `<meta name="description">` 태그 내용을 손금 테스트 관련 내용으로 업데이트
   - `<meta name="apple-mobile-web-app-title">` 태그를 "성격테스트"에서 "손금테스트"로 변경

3. **client/public/manifest.json**:
   - "name"을 "성격 분석 테스트 센터"에서 "손금 테스트"로 변경
   - "short_name"을 "성격테스트"에서 "손금테스트"로 변경
   - "description"을 손금 테스트 관련 내용으로 업데이트

### 2. 빌드 및 배포
1. 변경사항을 Git에 커밋
2. `npm run build`로 앱 빌드
3. `npm run deploy`로 GitHub Pages에 배포

## 기술적 세부사항

### 1. 메타데이터 일관성
모든 관련 파일에서 앱 이름, 설명 등을 일관되게 유지하는 것이 중요합니다. 특히 다음 항목들이 일치해야 합니다:
- 웹 페이지 제목 (`<title>` 태그)
- 앱 이름 (manifest.json의 "name" 필드)
- 앱 짧은 이름 (manifest.json의 "short_name" 필드)
- 앱 설명 (메타 태그와 manifest.json의 "description" 필드)

### 2. GitHub Pages 배포 프로세스
프로젝트에서는 gh-pages 패키지를 사용하여 GitHub Pages에 배포하고 있습니다:
- `predeploy` 스크립트: `npm run build` 실행
- `deploy` 스크립트: `gh-pages -d dist/public` 실행
- 이 과정에서 빌드된 파일들이 gh-pages 브랜치에 푸시됨

### 3. 서비스 워커 설정
PWA 기능을 위한 서비스 워커 등록 경로가 `/PalmReading/sw.js`로 설정되어 있어 GitHub Pages의 베이스 경로와 일치합니다.

## 결론 및 권장사항
1. **메타데이터 관리**: 앱의 메타데이터를 일관되게 유지하고, 프로젝트 변경 시 모든 관련 파일을 업데이트해야 합니다.
2. **배포 전 확인**: 배포 전에 빌드 결과물의 메타데이터가 올바른지 확인하는 과정이 필요합니다.
3. **GitHub Actions 활용**: 향후에는 GitHub Actions를 통한 자동화된 배포 프로세스를 구축하여 일관성을 유지하는 것이 좋습니다.
4. **CNAME 파일 관리**: GitHub Pages 배포 시 CNAME 파일이 올바르게 설정되었는지 확인해야 합니다.
